We needed to structure our apps

  • Create organized file structure
  • Manage dependencies and libraries
  • Provide architecture organization
  • Allow usage of objective patterns

Angular wasn't the first to solve this problem

Predecessors:

  • BackboneJS/Ember (MVC)
  • ExtJS (MVVM)
  • Marionette/Chaplin (5-tier)
  • Flux (6-tier)

But came with simplest approach

AngularJS (as for 1.3 *V*)

  • You care how you provide your data
  • You bind the data to view the way you like
  • You add structure to the app the way you need
  • We make your development fastest and easies possible

* Whatever

That became very popular

  • Easiest learn-to-go curve
  • Extended HTML syntax
  • Reduced the amount of code in JS
  • Declarative in-template logic implementation
  • Faster development-production cycle

Templates

HTML code augmented with directives, expressions and filters.

Module

Container for the different parts of app.

Sample module with dependency injection


angular
  .module('app.module', ['app.module.submodule_1','app.module.submodule_2'])

  .config(function ($stateProvider) {
    $stateProvider.state('app.module', {
      abstract: true,
      data: {
        permissions: {
          only: ['user'],
          redirectTo: 'auth.login'
        }
      }
    });
  });

  .run(function(greeter, user) {
    greeter.localize('Hello', user);
  });

Controller

Controller is a JavaScript constructor that is used to augment the Angular Scope.

Usage:

  • Set up the initial state of the $scope object.
  • Add behavior to the $scope object.

Sample controller


module.controller('GenderController', ['$scope', function($scope) {
  $scope.gender = undefined;

  $scope.clickMale = function() {
    $scope.gender = 'male';
  };

  $scope.clickFemale = function() {
    $scope.gender = 'female';
  };

  $scope.clickGender = function() {
    $scope.gender = 'gender';
  };
}]);

Directives

The heart and the power of Angular.

Marker on a DOM element that attach a specified behavior to that DOM element or even transform the DOM element and its children

Sample directive


app.directive('sampleDirective', function () {
  return {
    restrict: 'EA', //E = element, A = attribute, C = class, M = comment
    scope: {
      //@ reads the attribute value, = provides two-way binding, & works with functions
      title: '@'
    },
    template: '
{{ myVal }}
',
    templateUrl: 'mytemplate.html',
    controller: controllerFunction, //Embed a custom controller in the directive
    compile: function (element, attrs) { } // DOM manipulation
    link: function (scope, element, attrs) { } //DOM manipulation
  }
});

Filters

Selects a subset of items from array and returns it as a new array.

Can be used in JS and HTML code.

{{ filter_expression | filter : expression : comparator}}
$filter('filter')(array, expression, comparator)

Sample filter



<input ng-model="searchText">
<tr ng-repeat="friend in friends | filter:searchText">
  <td>{{friend.name}}</td>
  <td>{{friend.phone}}</td>
</tr>

Services

Value/Constant/Factory/Service/Provider

Objects whose API is defined by the developer writing the service.

Sample factory


app.factory('BaseAPI', function () {
  function API(model) {
    if (_.isUndefined(model)) {
      throw 'Argument "model" must be defined';
    }

    this.model = model;
  }

  API.prototype.fetch = function () {
    return Project
      .$collection()
      .$fetch()
      .$asPromise()
      .catch(handleError);
  });

  function handleError(){ /***/}

  return API;
});

Sample service


app.service('UserAPI', function (BaseAPI, User, $log, $rootScope) {
  var api = new BaseAPI(User);

  api.login = function login(loginData) {
  return User
    .$build(loginData)
    .$login()
    .$asPromise()
    .then(function (user) {
    // Make user, customers accessible globally
    $rootScope.user = user;
    return user;
  })
  .catch(function (response) {
    $log.error('Could not login the user', response);
  });

  return api;
});

Angular for and against

  • Easy to start, but requires programming knowledge to deeply understand
  • Requires pure JavaScript background to maintain larger app
  • Can be awfully slow
  • Has few architectural mistakes
  • Without selection of libraries doesn't shine
  • Framework will drastically change in 2.0
  • Very first component oriented architecture
  • 2x times faster development in comparison to other MV-like solutions
